How to use simulator

Choose the type of predictor you want to simulate. Click here for more information in our documentation.

To simulate a branch prediction, first you need a file with a sequence of addresses and branches decisions, for example, b77a8a3a T. After that, you need to choose the operating parameters, which in BHT are three: the # of indexes determines the number lines of the table; Local history size set the number of last local (in the index) decisions that will be saved to make a prediction; and Inicial value is the first prediction that will be made. GHT has one last parameter, Global history size is the number of global branch decisions (n parameter quoted in the documentation).

Now, the Simulate button needs to be clicked to appear the table and more six buttons:

  • Run fastly, Run slowly and Next step all initiate the the prediction, the difference is that the first two of them run the application on and on, while the other one runs just one step. Moreover they can be used to continue the prediction when it’s stopped;
  • Previous step moves backward one step;
  • Stop can be used when the application is running for itself and stops on the actual iteration;
  • Reset stops the application and goes to initial state of the table.

Finally, the Details field shows informations about the iteration number, address that the prediction was done and his index, and the predictor’s precision – ratio between the total of correct predictions and number of iterations.
